# История создания проекта RAG MCP Server

## Начало работы

1. **Изучение задачи** - Проанализирован файл task.md с требованиями к проекту
2. **Создание структуры** - Разработана базовая структура проекта

## Проблемы и решения

### Проблема 1: Зависимости Python
- **Описание**: Python не запускался, не хватало библиотек
- **Решение**: Созданы файлы `requirements.txt` и `requirements-dev.txt`

### Проблема 2: Несовместимость версий
- **Описание**: Python 3.14 несовместим с pydantic v1, который используется в chromadb
- **Решение**: Выбран вариант использования Docker или venv

### Проблема 3: Docker Compose
- **Описание**: Проблемы с запуском Docker после ряда правок
- **Решение**: Переключен MCP с STDIO на HTTP

### Проблема 4: Подключение к IDE
- **Описание**: MCP не работал в плагинах CLINE и Continue.dev для VS Code
- **Решение**: Создана утилита `mcp_client.py` для ручного вызова команд

### Проблема 5: Загрузка документов
- **Описание**: Ошибки загрузки документов
- **Решение**: Исправлены AI

### Проблема 6: Кодировка файлов
- **Описание**: Проблемы с кодировкой в файлах с русским текстом
- **Решение**: Русский язык лучше не использовать с малыми моделями (phi3:mini)

### Проблема 7: Функция ask_question()
- **Описание**: Функция не работала
- **Решение**: Найдено, что для векторизации использовались разные модели на вход и выход. Исправление моделей и возврат ранжировки найденных чанков привели к осмысленным ответам.

### Проблема 8: LangGraph Corrective RAG (недавно исправлено)
- **Описание**: Граф был линейным без условных переходов и retry-циклов
- **Решение**: Обновлён `src/rag/graph.py` - добавлены условные переходы и подключены узлы `hallucination_check` и `broaden_query`

### Проблема 9: Query Rewriting (недавно исправлено)
- **Описание**: Query rewriting был отключён, использовался оригинальный вопрос
- **Решение**: Включён `rewrite_query_node` - теперь использует LLM для переформулирования запроса
- AI заодно структурировал report.md, пусть так и останется, так красивше

### Проблема 10: Docker с подключением к Ollama на хосте
- **Описание**: Требовалось запустить Docker с подключением к Ollama на host-машине
- **Решение**: Добавлено автоматическое определение окружения в config.py - в Docker используется host.docker.internal, на хосте - localhost

### Проблема 11: Дефолтная директория для индексации
- **Описание**: Функция index_folder требовала явного указания пути к директории
- **Решение**: Добавлено значение по умолчанию sample_docs, автоматически определяется путь для Docker и хоста

## Используемые технологии
- **LLM**: minimax-m2.5 (ollama) для генерации кода
- **Векторная БД**: ChromaDB
- **Фреймворк**: LangGraph, LangChain
- **MCP**: FastMCP
